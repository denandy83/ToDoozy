## Story #39: Project Collaboration — 2026-04-01

### What was built
Full project collaboration feature: sharing personal projects via Supabase, invite links, real-time sync, member management, notifications, and task assignment.

### Architecture decisions
- **SyncService in renderer**: Since the authenticated Supabase client lives in the renderer process, all Supabase interactions (write-through, Realtime subscriptions, invite management) are handled there. SQLite operations stay in main via IPC.
- **Hybrid sync model**: Personal projects stay local-only. Shared projects write-through to both SQLite and Supabase Postgres. Offline edits queue locally in `sync_queue` table.
- **Labels sync by name**: Labels are personal (each user has own colors). Only label names are synced via `label_names` JSON array on shared_tasks. Missing labels are auto-created locally.
- **Supabase RLS**: All shared tables have Row Level Security enforcing project membership via `shared_project_members`.
- **Deep links**: `todoozy://invite/{token}` registered via `app.setAsDefaultProtocolClient()` with `open-url` handler on macOS.
