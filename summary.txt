## Stories Reviewed

### Story 1-5 (Scaffold, DB, Repos, IPC, Stores) — PASSED
No user-visible issues.

### Story 6 (Auth / Login) — FIXED
- BUG: Google OAuth worked but after login the app showed a black screen.
  Root cause: Zustand 5's `create()` uses `useSyncExternalStore` without selector caching.
  Any selector returning a new reference caused infinite re-render loops.
  FIX: Switched all 9 stores to `createWithEqualityFn` with `shallow` equality.
  Also added `use-sync-external-store` peer dep.
- BUG: Default "Personal" project not visible after first login.
  Root cause: `ensureDefaultProject` created the project but never added a `project_members` row.
  `getProjectsForUser` JOINs on `project_members`, so the project was invisible.
  FIX: Added `addMember` call in `ensureDefaultProject` and `NewProjectModal`.
  Also added backfill for existing users on login.

### Story 7 (Projects & Statuses) — FIXED
- Removed "Danger Zone" label from project settings (too alarming, "Delete Project" is enough)
- Status reordering: replaced non-functional grip handle with @dnd-kit drag-and-drop
- Added X close button to shared Modal component

### Story 8 (Task CRUD) — PASSED
- Status cycling works as designed (wraps around). User initially thought tasks "disappeared"
  when clicking Done status — they just moved to the "Not Started" section.

### Story 9 (Views & Sidebar) — FIXED

### Story 10 (Detail Panel) — PARTIALLY REVIEWED, IN PROGRESS
Fixed so far:

- Label picker positioning: opens within task list container, doesn't overlap detail panel

- Snooze: extracted shared utility (shared/utils/snooze.ts), preserves time on due date
- Snooze context menu now also preserves time (was using separate `addDays` that stripped time)
